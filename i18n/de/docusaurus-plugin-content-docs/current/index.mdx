# Was ist coasti?

**coasti liefert BI im Produktformat.**

Die meisten BI-Landschaften wachsen organisch: Dashboards liegen in einem Tool, Transformationen in einem anderen Repository, Ingestion-Skripte irgendwo dazwischen.
Soll derselbe Bericht für ein zweites Team oder einen weiteren Kunden laufen, wird alles von Hand neu zusammengesetzt.


coasti bündelt das: Alles, was ein Bericht braucht — von den Rohdaten bis zum fertigen Dashboard — wird in ein einziges, versioniertes, installierbares **Produkt** gebündelt.
Installieren, ein paar Fragen beantworten, fertig ist eine vollständige, lauffähige BI-Lösung.
Aktualisiert wird sie wie ein Software-Paket.

## Aufbau eines coasti-Produkts

![Der coasti-Stack: Ingestion, Storage, dbt-Transformationen, Orchestrierung und Superset-Frontend-Inhalte, gebündelt in einem Produkt](https://github.com/user-attachments/assets/acf96895-9d3b-412e-a012-6a5ef85492dc)

Eine coasti-Produktdefinition besteht aus:

| Schicht | Inhalt |
|---|---|
| **Frontend-Content** | Alle Superset-Objekte des Produkts: Dashboards, Charts, Datasets und mehr |
| **Orchestration** | Eine schlanke Python-Pipeline (aktuell auf Basis von [Hamilton](https://github.com/dagworks-inc/hamilton)), die alle Schichten verbindet |
| **Transformation** | Die Transformationslogik in dbt: Engine-agnostisch, läuft auf DuckDB, Postgres oder allem, was dbt unterstützt |
| **Ingestion** | Wie Rohdaten in das Produkt gelangen, abgelegt als Parquet-Dateien |
| **Storage** | Lokales Dateisystem oder S3: freie Wahl |

Weil jede Schicht deklarativ im Produkt definiert ist, ist es vollständig portabel: Dasselbe Produkt läuft auf dem Laptop mit DuckDB und lokalen Dateien genauso wie in der Produktionsumgebung mit Postgres und S3.

## Der Open-Source-Stack

coasti besteht aus einer Reihe öffentlicher Repositories, die zusammenspielen:

- **[coasti_installer](https://github.com/coasti-org/coasti_installer)**: installiert vollständige Produkte (dbt, Ingestion, Orchestrierung, Frontend-Inhalte) auf der eigenen Infrastruktur. Antwortdateien machen das Deployment einfach und reproduzierbar, eine integrierte Update-Funktion hält installierte Produkte aktuell.
- **[superset_io](https://github.com/coasti-org/superset_io)**: exportiert und importiert alle Frontend-Inhalte (Dashboards, Datasets, Charts, …) von und nach Superset, sodass die Frontend-Ebene eines Produkts wie Code versioniert werden kann.
- **[superset_docker](https://github.com/coasti-org/superset_docker)**: Superset als produktionsreifer Docker-Stack.
- **[superset_lucide_extension](https://github.com/coasti-org/superset_lucide_extension)**: eine Lucide-Icon-Erweiterung für Superset (wird künftig standardmäßig Teil unseres Docker-Images).

## Ein offenes Produkt-Ökosystem

Das Produktformat ist offen — **jeder kann eigene coasti-Produkte erstellen und teilen.**
Ein Produkt ist einfach ein Git-Repository mit definierter coasti-Struktur; Bauen, Versionieren und Verteilen funktionieren also mit den gewohnten Werkzeugen der Softwareentwicklung.

Wie ein vollständiges Produkt aussieht, zeigt unser Open-Source-Beispiel:

- **[linkfish_genesis_stats](https://github.com/coasti-org/linkfish_genesis_stats)**: ein komplettes Produkt auf Basis amtlicher Statistikdaten (GENESIS/Destatis und Regionaldatenbank), vom automatischen Datendownload über dbt-Modelle bis zu fertigen Superset-Dashboards. Weitere Open-Source-Produkte folgen.

Neben den Open-Source-Produkten bieten wir fertige kommerzielle Produkte für bestimmte Anwendungsbereiche an — aufgebaut auf demselben offenen Format und genauso installierbar.

## In der Praxis bewährt

coasti ist kein Experiment auf der grünen Wiese.
Der Stack hat sich in Produktionsumgebungen bei mehreren Kunden bewährt — vor allem in öffentlichen Verwaltungen, die ihr tägliches Berichtswesen darauf betreiben.
