# Installation

We can get from zero to a working report in superset in about 10 minutes.
The installation requires some dependencies and the use of command-line tools.
This tutorial walks you through the process, shows the commands and explains each step.

Let's start by gathering the needed dependencies.

:::note
For production deployments, we set up coasti so that superset runs on a different machine than the transformations. This way, expensive computations of the data-pipeline do not affect the user experience. For simplicity, we put both on the same host in this tutorial.
:::

## Get the coasti installer

The coasti installer is a small python package that wraps around existing open-source tools.
It allows us to create and manage coasti projects, set up version control, and to add or update content.

The easiest way to get the coasti installer is through [uv](https://docs.astral.sh/uv/getting-started/installation/):

<Tabs groupId="os">
	<TabItem value="macos-linux" label="macOS / Linux" default>

        <Terminal>
        ```bash
        curl -LsSf https://astral.sh/uv/install.sh | sh

        uv tool install coasti
        ```
        </Terminal>

	</TabItem>

	<TabItem value="windows" label="Windows">

        <Terminal>
        ```powershell
        powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

        uv tool install coasti
        ```
        </Terminal>

	</TabItem>
</Tabs>

## Initialize a coasti project

All products, content and metadata that you use with coasti live in a central folder, a _coasti project_.
Projects are initialized via `coasti init` and you need to provide a folder path to use.

We recommend to use an absolute and simple path like `/coasti`.
This makes it a bit easier to map configs into containers, where it helps to use the same paths inside and out.

## Install Apache Superset

With the coasti project set up, we can add superset as a product.
We provide a batteries-included [docker image](https://github.com/coasti-org/superset_docker.git) that streamlines the installation.

First, coasti asks a few questions on how to get the product.
This lets you choose which branch to use (products are just git repositories) or how to gain access, if your product is in a private repo. Because superset is in a public repo, we dont need authentication.

<Terminal copyOnlyPrompts highlightPrompts typewriter typewriterSpeed={40}>
```bash
[prompt] cd /coasti
[prompt] coasti product add https://github.com/coasti-org/superset_docker.git

🎤 Install location:
[highlight]    products/superset_docker

🎤 Version control reference (git branch, tag or commit)
[highlight]    main

Added superset_docker in products.yml

🎤 Do you want to install superset_docker now?
[highlight]    No
```
</Terminal>


After adding superset as a product to coasti's metadata, we are asked to _install it_.
We can either do this directly (type `y` or `↵`) or install later via `coasti product install`.

This will clone the repository, and guide us through the questions needed to configure superset.
For most questions, choosing the default by pressing `↵` is fine.

You can later check the details based on your answers in `/coasti/products/superset_docker/config/.env`.

Now, we can open a browser and navigate to http://localhost:8088 to login with the credentials we entered earlier.

With the `up` command we just used, we launched the superset-app container and attached to it. This directly shows all output in the Terminal.
To stop the container, and return to our shell, we can press `ctrl + c`.

If we want the superset containers to run in the background, we can start detached by using `docker compose up -d`

<Terminal>
```bash
docker compose -f ./docker/docker-compose.yml --env-file ./config/.env up -d superset-app
```
</Terminal>
